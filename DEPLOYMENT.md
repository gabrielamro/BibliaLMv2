
# 🚀 Roteiro de Migração: BíbliaLM (MVP para Produção)

Este documento é o guia definitivo de engenharia para migrar o BíbliaLM de uma arquitetura *Client-Side/Firebase (NoSQL)* para uma arquitetura *Server-Side/PostgreSQL (Supabase)*, visando escala infinita, performance e redução drástica de custos com IA via RAG (Retrieval-Augmented Generation).

---

## 1. Arquitetura de Produção

### Stack Tecnológica
*   **Banco de Dados:** Supabase (PostgreSQL 15+).
    *   *Por que:* Relacional (SQL), ACID, Open Source, sem lock-in, suporta Vetores (IA) nativamente.
*   **Backend (Lógica):** Supabase Edge Functions (Deno/Node).
    *   *Por que:* Centraliza a lógica de negócios (XP, Assinaturas, IA), impedindo fraudes no frontend.
*   **IA & Busca:** pgvector (Extensão do Postgres).
    *   *Por que:* Permite busca semântica ("versículos sobre ansiedade") dentro do banco, eliminando a necessidade de Pinecone/Algolia.
*   **Storage:** Supabase Storage (S3 compatible).
    *   *Por que:* Migração fácil do Firebase Storage.

### Vantagens da Migração
1.  **Custo Zero de Leitura:** Buscar um capítulo da Bíblia no Postgres é virtualmente grátis e instantâneo, ao contrário do Firebase que cobra por leitura de documento.
2.  **Busca Inteligente:** O Postgres tem Full-Text Search e Busca Vetorial nativos.
3.  **Relacionamentos:** Consultas complexas ("Trazer todos os posts dos membros da minha célula ordenados por data") são triviais em SQL e custosas em NoSQL.

## 2. Modelagem do Banco de Dados (Schema SQL)

Copie e cole este script no **SQL Editor** do Supabase para criar sua estrutura de banco de dados completa.

```sql
-- 1. HABILITAR EXTENSÕES ESSENCIAIS
create extension if not exists vector;       -- Para IA e Embeddings
create extension if not exists pgcrypto;     -- Para criptografia e UUIDs
create extension if not exists pg_trgm;      -- Para busca textual rápida (fuzzy search)

-- 2. PERFIS DE USUÁRIO (Sincronizado com Auth)
create table public.profiles (
  id uuid references auth.users not null primary key,
  username text unique not null,
  display_name text,
  photo_url text,
  bio text,
  city text,
  state text,
  credits int default 0,
  lifetime_xp int default 0,
  subscription_tier text default 'free', -- free, bronze, silver, gold, pastor
  subscription_status text default 'active',
  created_at timestamptz default now()
);

-- 3. BÍBLIA SAGRADA (O Coração do App)
create table public.bible_books (
  id text primary key, -- ex: 'gn', 'jo'
  name text not null,
  testament text not null -- 'old', 'new'
);

create table public.bible_verses (
  id bigint generated by default as identity primary key,
  book_id text references public.bible_books(id),
  chapter int not null,
  verse int not null,
  text text not null,
  -- O embedding é a "versão matemática" do texto para a IA
  embedding vector(1536) 
);

-- Índices para performance extrema
create index on public.bible_verses (book_id, chapter, verse);
create index on public.bible_verses using ivfflat (embedding vector_cosine_ops) with (lists = 100);

-- 4. CONTEÚDO GERADO PELO USUÁRIO (UGC)

-- Estudos e Sermões
create table public.studies (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.profiles(id) not null,
  title text not null,
  content text,
  source_type text, -- 'sermon', 'devotional', 'chat'
  is_public boolean default false,
  likes_count int default 0,
  created_at timestamptz default now()
);

-- Anotações Pessoais
create table public.notes (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.profiles(id) not null,
  book_id text references public.bible_books(id),
  chapter int,
  verse int,
  content text not null,
  created_at timestamptz default now()
);

-- 5. SOCIAL & COMUNIDADE

-- Igrejas
create table public.churches (
  id uuid default gen_random_uuid() primary key,
  name text not null,
  slug text unique not null,
  location_city text,
  location_state text,
  logo_url text,
  created_at timestamptz default now()
);

-- Células / Grupos
create table public.cells (
  id uuid default gen_random_uuid() primary key,
  church_id uuid references public.churches(id),
  name text not null,
  leader_id uuid references public.profiles(id),
  created_at timestamptz default now()
);

-- Vínculo Membro -> Igreja/Célula
create table public.memberships (
  user_id uuid references public.profiles(id) primary key,
  church_id uuid references public.churches(id),
  cell_id uuid references public.cells(id),
  role text default 'member', -- member, leader, admin
  joined_at timestamptz default now()
);

-- Feed do Reino (Posts)
create table public.posts (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.profiles(id) not null,
  content text,
  image_url text,
  type text default 'general',
  likes_count int default 0,
  comments_count int default 0,
  church_id uuid references public.churches(id), -- Null se for post global
  created_at timestamptz default now()
);

-- Pedidos de Oração
create table public.prayer_requests (
  id uuid default gen_random_uuid() primary key,
  user_id uuid references public.profiles(id) not null,
  content text not null,
  target_type text, -- 'global', 'church', 'cell'
  target_id uuid,   -- ID da igreja ou célula
  intercessors_count int default 0,
  created_at timestamptz default now()
);

-- 6. FUNÇÃO RPC PARA BUSCA SEMÂNTICA (RAG)
-- Permite perguntar: "Versículos sobre medo" e receber os IDs exatos.
create or replace function match_verses (
  query_embedding vector(1536),
  match_threshold float,
  match_count int
)
returns table (
  id bigint,
  book_id text,
  chapter int,
  verse int,
  text text,
  similarity float
)
language sql stable
as $$
  select
    v.id,
    v.book_id,
    v.chapter,
    v.verse,
    v.text,
    1 - (v.embedding <=> query_embedding) as similarity
  from public.bible_verses v
  where 1 - (v.embedding <=> query_embedding) > match_threshold
  order by v.embedding <=> query_embedding
  limit match_count;
$$;
```

## 3. Segurança (Row Level Security - RLS)

Segurança a nível de linha é crucial. No Supabase, isso substitui as Security Rules do Firebase.

```sql
-- Habilitar RLS em todas as tabelas críticas
alter table public.profiles enable row level security;
alter table public.studies enable row level security;
alter table public.notes enable row level security;

-- Políticas de Acesso (Exemplos)

-- Perfis: Qualquer um lê, só o dono edita.
create policy "Public profiles are viewable by everyone"
  on public.profiles for select using ( true );

create policy "Users can update own profile"
  on public.profiles for update using ( auth.uid() = id );

-- Notas: Só o dono vê e edita (Privacidade Total).
create policy "Users can perform all on own notes"
  on public.notes for all using ( auth.uid() = user_id );

-- Estudos: Dono vê os seus, todos veem os públicos.
create policy "Users can see own studies"
  on public.studies for select using ( auth.uid() = user_id );

create policy "Public studies are viewable by everyone"
  on public.studies for select using ( is_public = true );
```

## 4. Estratégia de Migração (Passo a Passo)

### Fase 1: Importação da Bíblia (Crucial)
Você já possui o script `scripts/generate_full_bible.js` que gera o JSON. O próximo passo é enviar isso para o Supabase e gerar os Embeddings.

1.  Rode o script: `node scripts/generate_full_bible.js`
2.  Crie um novo script `scripts/seed_supabase.js` (usando `supabase-js`) para ler esse JSON e inserir na tabela `bible_verses`.
3.  **Embeddings:** Para cada versículo inserido, chame a API da OpenAI (model `text-embedding-ada-002` ou `3-small`) para gerar o vetor e salvar na coluna `embedding`. Isso transforma seu banco em uma IA.

### Fase 2: Autenticação e Dados de Usuário
O Supabase tem ferramentas para importar usuários do Firebase Auth.
1.  Exporte usuários do Firebase (Console > Auth > Users > Export).
2.  Importe no Supabase Auth.
3.  Escreva um script para ler o Firestore (`users` collection) e migrar os documentos para a tabela `public.profiles`.

### Fase 3: Backend & Edge Functions
Para a gamificação segura (XP), crie uma Edge Function no Supabase.

**Exemplo de Edge Function (`add_xp`):**
```typescript
// supabase/functions/add_xp/index.ts
import { serve } from "https://deno.land/std@0.168.0/http/server.ts"
import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'

serve(async (req) => {
  const { action } = await req.json()
  // Lógica de validação...
  
  const xpAmount = action === 'read_chapter' ? 20 : 5;
  
  // Atualiza no banco direto (seguro)
  const { data, error } = await supabase.rpc('increment_xp', { amount: xpAmount });
  
  return new Response(JSON.stringify({ new_xp: data }), { headers: { 'Content-Type': 'application/json' } })
})
```
